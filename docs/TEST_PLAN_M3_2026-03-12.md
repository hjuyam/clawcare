# ClawCare M3 Test Plan — 2026-03-12

## Scope
M3 “真实落盘语义”验收（apply/rollback/safe mode）。

## P0 — 核心验收

### Apply
1. **Base version guard**
   - 前置：读取 `current_version` 与 `base_version`。
   - 操作：使用**不匹配**的 `base_version` 执行 apply。
   - 期望：请求被拒绝（明确错误/冲突状态）；**不产生落盘**；`current_version` 不变；`versions` 列表不新增；审计/运行记录标记失败。

2. **Apply 成功路径**
   - 前置：`base_version == current_version`。
   - 操作：执行 apply（带新 config）。
   - 期望：`current_version` 变为新版本；`versions` 列表新增 entry（包含时间戳/操作者/摘要）；落盘 snapshot 文件存在且可读取；配置内容与提交一致。

3. **Apply 版本与快照一致性**
   - 前置：完成一次成功 apply。
   - 操作：读取 `current config` 与 snapshot。
   - 期望：`current config` 与 snapshot 内容一致（字节级或结构级对比）；版本号与 snapshot 元数据对应。

### Rollback
1. **回滚到目标版本**
   - 前置：至少存在两个历史版本。
   - 操作：选择历史版本执行 rollback。
   - 期望：`current_version` 切换为目标版本；`current config` 与目标版本 snapshot 一致；`versions` 列表记录 rollback 事件/新 entry（如设计要求）。

2. **Rollback 审计**
   - 操作：执行 rollback。
   - 期望：审计事件存在（action=rollback/target_version/actor/result）。

### Safe Mode
1. **Safe Mode 拦截 apply**
   - 前置：开启 safe mode。
   - 操作：尝试 apply。
   - 期望：请求被拒绝；策略日志中存在 **policy deny** 记录（含原因）；`current_version` 不变；无新 snapshot。

2. **Safe Mode 拦截 rollback**
   - 前置：开启 safe mode。
   - 操作：尝试 rollback。
   - 期望：请求被拒绝；策略日志中存在 **policy deny** 记录；`current_version` 不变。

## P1 — 补充/健壮性

1. **重复 apply 幂等性**
   - 前置：对同一 config 连续 apply。
   - 期望：第二次 apply 若内容完全一致应表现为 no-op 或记录新版本但内容一致（依产品设计）；不应导致 snapshot 缺失或版本跳变异常。

2. **Rollback 到最新版本**
   - 操作：对当前版本执行 rollback。
   - 期望：合理拒绝或 no-op；审计记录明确。

3. **审计与版本列表一致性**
   - 操作：对 apply/rollback 各执行一次。
   - 期望：审计事件与 `versions` 列表条目一一对应（时间/actor/版本一致）。

4. **异常落盘保护**
   - 操作：模拟落盘失败（权限/磁盘满/写入异常）。
   - 期望：apply/rollback 失败且回滚到安全状态；不会出现 `current_version` 变化但 snapshot 缺失的“半成功”状态。
